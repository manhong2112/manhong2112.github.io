    最近在做 Mimikko-Mod, 想加個 Xposed 的 Annotation 自動 hook // 就是挖坑給自己跳

    部分內容從簡, 因為我忘了

    首先呢, 照着以前 Android apt 的教程（畢竟我根本沒找到 Kotlin + Android + Annotation processor 的詳細教程), 弄了兩個 module, annotation和processor…

    root/settings.gradle

    include ':annotation'
include ':processor'

include ':app'

    annotation/build.gradle

    // kotlin 支持, 略

    processor/build.gradle

    // 略
dependencies {
    // 略
    compileOnly project(":annotation")

    compileOnly 'com.google.auto.service:auto-service:1.0-rc4'
    kapt 'com.google.auto.service:auto-service:1.0-rc4'
}

    app/build.gradle

    apply plugin: 'kotlin-kapt'
// 略
dependencies {
    // 略
    compileOnly project(":annotation")

    compileOnly project(":processor")
    kapt project(":processor")
}

    然後真正弄個 annotation class..

    annotation/../Hook.kt

    // 略
@Target(AnnotationTarget.FUNCTION)
@Retention(AnnotationRetention.SOURCE)
annotation class HookAfter(val method: String, val argsType: Array<String> = [], val priority: Int = 50)

    然後 annotation processor…

    processor/../HookProcessor.kt

    package me.manhong2112.xposed.processor

import com.google.auto.service.AutoService
import javax.annotation.processing.*
import javax.lang.model.SourceVersion
import javax.lang.model.element.TypeElement

@AutoService(Processor::class)
@SupportedSourceVersion(SourceVersion.RELEASE_8)
@SupportedAnnotationTypes("me.manhong2112.xposed.annotation.HookAfter")
class HookProcessor: AbstractProcessor() {
   override fun process(annotations: MutableSet<out TypeElement>?, roundEnv: RoundEnvironment?): Boolean {
      throw Throwable("process")
   }

   @Synchronized override fun init(processingEnv: ProcessingEnvironment?) {
      throw Throwable("init")
   }
}
    e: [kapt] An exception occurred: java.lang.Throwable: init

    嗯..至此一切正常, 因為不正常的都跳過了

    然後要測試@SupportedAnnotationTypes, 於是註解掉throw Throwable("init")…

    Annotation processors must be explicitly declared now.  The following dependencies on the compile classpath are found to contain annotation processor.  Please add them to the annotationProcessor configuration.
  - processor.jar (project :processor)
Alternatively, set android.defaultConfig.javaCompileOptions.annotationProcessorOptions.includeCompileClasspath = true to continue with previous behavior.  Note that this option is deprecated and will be removed in the future.
See https://developer.android.com/r/tools/annotation-processor-error-message.html for more details.

    喵喵喵??? 這個不應該是 @AutoService 處理的嗎?
    翻了翻processor/build, 不對啊, javax.annotation.processing.Processor 有生成啊, me.manhong2112.xposed.processor.HookProcessor

    也在裡面試着複製出來塞到 main 裡, 錯誤依舊

    求助Google

    SO 某用戶指出可以

    kapt project(":compiler")
annotationProcessor project(":compiler")
compileOnly project(":compiler")

    嗯…管用就行了~

    最後確保 process 管用, 隨便找一個 method 加上 @HookAfter("")

    e: [kapt] An exception occurred: java.lang.Throwable: process
    至此 annotation processor 終於是能用了
