0x0

最近撿回了hexo這玩意，準備繼續寫點東西，然後又開始了無盡的主題配置嗯……
在想着：夢醬啊夢醬你不能這麼下去啊 的時候
突然想起，既然弄的是hexo，那就來寫點hexo的東西吧
所以，這篇就來記一下hexo主題的框架

btw，這裡只會說主題的框架，不會教你怎麼用ejs/js/html，如果你有需要請出門右轉官方教程。

0x1 Template

hexo的主題格式如下

themes_root
 |-- languages/ // 多語言
 |-- layout/ // 放主題的模板文件, 夢開始的地方
 |   |-- layout.ejs // 據說一切頁面都會先經過layout.ejs
 |   |-- index.ejs // 見下文...
 |-- source/ // 放置 css/scss/sass/js 等資源的地方, 會 被渲染然後/直接 複製到 public資料夾裡
 |-- scripts/ // 給Hexo的擴展用的, helper函數甚麼的
 |-- _config.yml // 主題設置檔, 在layout

0x2 index.ejs – 在夢開始之前

在說layout.ejs咱先說說這個index.ejs

在文章的frontmatter可以設置layout的值，不寫的話會使用hexo的_config.yml的default_layout (預設值為post)

而 hexo 套用模板依照layout值以 tag|category->archive|page|post->index這種方式 fallback

也就是說，只要你有index.ejs，你就可以有一個正常運作的主題模板了，然後其他都是ejs/swig啥啥啥模板引擎和查hexo api的工作，去看看官方doc就好。好的此文完結撒花(?

0x3 layout.ejs

好吧好吧不能這麼水…再說說layout.ejs吧，一般來說主題都會用到layout.ejs來進行填充

但首先，layout.ejs這玩意不是必須的，我被無數文章坑了導致我一直沒搞懂這玩意是干甚麼的…

因為這個流程上是

  1. 渲染文章的 layout 對應的.ejs (比如index.ejs)
  2. 如果存在 layout.ejs 就把剛才渲染好的.ejs (比如index.ejs) 塞到 layout.ejs 裡並渲染layout.ejs
  3. 如果不存在的話就直接輸出 1 的結果

所以layout.ejs基本上是用來放所有頁面通用的框架的 (但這不是模板應該做的事情嗎嗯…)，比如這樣，放<!DOCTYPE html>放<html></html>，放一些js

<!-- layout.ejs -->
<!DOCTYPE html>
<html>
    <head></head>
    <%- body %>
</html>
<!-- index.ejs -->
<body>
    This is index page.
</body>
<!-- index.html -->
<html>
    <head></head>
    <body>
        This is index page.
    </body>
</html>

<%- body %>是ejs的語法，其中 body 是一個變量， <%- body %> 就會輸出 body 的內容，而body則放了剛才步驟 1 裡渲染的.ejs (比如index.ejs)，其他模板應該也一樣，但我沒試過。

0x? 後記

在寫完這篇文章後我開始着手我的主題，在一堆多得讓人看得眼花繚亂的模板引擎中選中了Jade (Pug)。然後…一整天都在研究這個那個怎麼實現，然後發現貌似根本實現不了，最後得寫長長一段的來代替嗯…

這語言最大的問題不是縮進語法，而是自創了一套語法，但又不夠完整，為了同時支援JavaScript/Pug/HTML etc.創造一堆語法，但卻又沒想周全/覺得沒有必要。比如要在多行的Pug裡插入多行HTML然後再在裡面插入多行Pug，你會發現根本無法實現。當然你總是可以把代碼塊拆開來做到，但只有兩三行的代碼也要拆出去這嗯…

最簡單的一個例子：

style.
   html {
      // 我到底要怎麼樣才可以在這裡插入多行Jade/Javascript ??
   }

你只能夠這樣：

style
   | html {
      // blabla
   | }
// 或者
style.
   html {
      #[+html-style]
   }
mixin html-style
   // blabla

可是如果很多很多行呢，一堆 | 嗎？每一段style都要拆出去？不麻煩嗎？多幾層怎麼辦？…

還有這語言有 . | #[] #{} !{} ，再混上js的 ${}，嗯…..我已經分不清哪個是哪個了…

總之我再用幾天看看吧…到時再換也不遲

Pug不適合寫複雜邏輯，他真的只是個加了糖的HTML
